# Eclipsing Binary Light Curve Simulation 🌌

A mathematical and astrophysical simulation tool written in Python to model, calculate, and plot the light curves of detached eclipsing binary star systems.

## 🧠 Astrophysical Context
An **Eclipsing Binary System** consists of two stars orbiting their common center of mass, where the orbital plane is aligned close to our line of sight ($i \approx 90^\circ$). As they periodically pass in front of each other, the total observed brightness drops, creating a light curve.
* **Primary Eclipse:** Occurs when the cooler (less luminous) star occults the hotter star, resulting in a deeper drop in flux.
* **Secondary Eclipse:** Occurs when the hotter star occults the cooler star, resulting in a shallower dip.

The geometric shape of these dips (V-shape vs. U-shape) and their depths allow astronomers to determine stellar radii, surface temperatures, and contact times.

## 💻 Mathematical & Geometric Modeling
The simulation accounts for:
1. **Orbital Mechanics:** Simulates uniform circular motion to project 2D coordinates $(x, y)$ of both stars onto the plane of the sky.
2. **Circle-Circle Overlap Geometry:** Implements analytical equations for intersecting chords and circular segments to calculate the exact occulted area at any given orbital phase.
3. **Pogson's Law:** Converts linear total flux into logarithmic relative magnitudes standard in astronomical photometry:
   $$m = -2.5 \log_{10}(\text{Flux})$$

## 🛠️ Features
- **Pure Python Implementation:** Built using `NumPy` for heavy geometric calculations and `Pandas` for data management.
- **Astronomically Accurate Plotting:** Generates a phase-folded light curve with an inverted Y-axis (as per astronomical convention where smaller magnitudes represent brighter states).
- **Contact Detection:** Dynamically switches between non-eclipse, partial eclipse, and total/annular eclipse scenarios.
